오라클 클러스터 검증 유틸리티(Cluster Verification Utility)
------------------------------------------------

### 1.개요

오라클 클러스터 검증 유틸리티(Cluster Verification Utility, CVU)는 Oracle Grid Infrastructure 또는 Oracle Real Application Clusters(Oracle RAC)의 설치, 업그레이드, 패치의 작업이 성공적으로 완료될 수 있도록 필요한 시스템 구성의 사전, 사후 점검을 지원하는 툴이다.

점검 대상

CVU는 디스크 공간, 메모리, 프로세스, 네트워크  및 기타 중요한 클러스터 구성 요소를 확인한다.

### 2.설치 준비

#### 스토리지 공간

실행 노드에 최소 30MB, 각 노드에 최소 25MB의 여유 공간(/tmp)이 필요하다.

#### 실행파일 위치

실행 파일은 그리드 인프라스트럭처가 설치될때 자동 설치되며, 최신 버전을 오라클 MOS에서 다운로드 할수 있다.

실행파일 위치 : $GRID_HOME/bin

#### 설치 계정

CVU 설치, 실행은 대부분 그리드 인프라스트럭처 계정(oracle)을 사용한다. 일부 점검은 root 계정이 필요하다.

#### SSH 구성

모든 클러스터 노드에 대해 SSH를 구성해야 한다. CVU 설치 때 SSH 구성을 완료하거나 클러스터의 모든 노드 간에 SSH 구성을 직접 완료 할 수도있다.


### 3. 실행 방법

#### 보고서 저장소 생성

검증 결과 보고서를 저장할 수 있는 별도 디렉토리를 만들어서 사용하면 편리하다.

```command
mkdir -p $ORACLE_BASE/cvu

```
#### CVU 실행

```command
cd $ORACLE_ASE/cvu
```

##### 시스템 설치 요건 검증

command : cluvfy comp sys

```command
cluvfy comp sys -allnodes -p crs -verbose > sys_crl.log    // crs 검증

cluvfy comp sys -allnodes -p database -verbose > sys_database.log   //database 검증

cluvfy comp sys -allnodes -p ha -verbose > sys_ha.log    //ha 검증
```
#### 공유 네트워크 및 스토리지 검증

command: cluvfy stage hwos

```command
cluvfy stage -post hwos -allnodes -verbose > hwos.log   //네트워크 검증
```
#### 시간 동기화 검증

command : cluvfy comp clocksync
```
cluvfy comp clocksync -allnodes -verbose > clocksync.log   //클러스터 시간 동기화
```
#### 관리자 권한 검증

command :cluvfy comp admprv

```
cluvfy comp admprv -allnodes -o user_equiv -fixup > admprv.log  // 관리자 권한 검증
```
#### 커넥테비티 검증

command : cluvfy comp {nodecon | nodereach}

```command
cluvfy comp nodecon -allnodes -verbose > nodecon.log   // 노드 연결 검증

cluvfy comp nodereach -allnodes -verbose > nodereach.log   // 노드 연결 가능성 검증
```
#### 노드간 소프트웨어 호환성(compatibility) 검증

command : cluvfy comp peer

```command
cluvfy comp peer -allnodes > peer.log    //호환성(compatibility) 점검
```

#### 클러스터웨어, ASM 컴포넌트 검증

command : cluvfy comp {asm | clumgr | crs | ocr | nodeapp | ohasd | scan | software | vdisk }

```command

cluvfy comp asm -allnodes -verbose > asm.log    //ASM 무결성

cluvfy comp clumgr -allnodes -verbose > clumgr.log   //클러스관리자 무결성

cluvfy comp crs -allnodes > crs.log   // CRS 무결성 검증

cluvfy comp ocr -allnodes -verbose > ocr.log  //OCR 무결성 검증

cluvfy comp nodeapp -allnodes -verbose > nodeapp.log // node application 존재 검증

cluvfy comp ohasd -allnodes -verbose > osdhd.log   // Oracle HA 서비스 데몬 검증

cluvfy comp scan -verbose > scan.log    // SCAN 구성 검증

cluvfy comp software -allnodes -verbose > grid_sw.log     // grid_sw 검증

cluvfy comp software -d $ORACLE_HOME -allnodes -verbose > db_sw.log  // DB_HOME 검증

cluvfy comp vdisk -allnodes -verbose > vdisk.log    //vdisk 검증

```

#### 설치 위치 검증

command : cluvfy stage {appcluster | hwos | dbcfg | dbinst | crsinst | hacfg | hodeadd}

```command

cluvfy stage -pre appcluster -allnodes > appcluster_pre.log   // 클러스터웨어 설치전 검증

cluvfy stage -post appcluster -allnodes -verbose > appcluster_post.log  // 클러스터웨어 설치후 검증

cluvfy stage -pre dbcfg -allnodes -d $DB_HOME > dbcfg_pre.log // RAC DB 설치 위치 점검

cluvfy stage -pre dbinst -allnodes > dbinst.log  // RAC DB 시스템 환경 점검

cluvfy stage -pre crsinst -allnodes > crsinst_pre.log // CRS 설치 전 검증

cluvfy stage -post crsinst -allnodes > crsinst_post.log   //  CRS 설치 후 검증

cluvfy stage -pre hacfg > hacfg_pre.log  // Oracle Resatrt 구성 위한 사전 로컬노드 검증

cluvfy stage -post hacfg > hacfg_post.log // Oracle Resatrt 구성 위한 사후 로컬노드 검증

cluvfy stage -pre nodeadd -n node_list    // RAC 노드 추가전 검증

cluvfy stage -post nodeadd -n node_list    // RAC 노드 추가후 검증

```

#### 삭제된 노드의 검증

command : cluvfy stage nodedel

```command
cluvfy stage -post nodedel -n node_list // node_list의 정상 삭제 여부 점검

```
